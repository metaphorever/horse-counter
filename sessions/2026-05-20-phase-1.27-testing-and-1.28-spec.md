## What shipped

### Bug fixes (PR #39, committed d9d59f2)
- **`/me/drafts` 500 server error** — `{{ url_for(\'poetry_editor\') }}` inside a JavaScript string in `me_drafts.html` caused a Jinja2 TemplateSyntaxError on every render. Fixed by using double-quoted arg: `{{ url_for("poetry_editor") }}`.
- **Draft modal blanks on re-open** — `openDraftModal()` always zeroed all fields. Fixed by adding `currentDraftMeta` state variable in `poetry.html`, populated on save success, used to pre-populate modal when re-opened on the same draft.
- **Touch scroll jank in search** — `.horse-tile` has `touch-action: none` globally (for drag support in the poem compose area), but search result chips also use `.horse-tile`. Fixed with `.search-results .horse-tile { touch-action: auto; }` scoped override in `style.css`.
- **Tag chip display issue** — defensive fix: added `position: relative` to `.tag-chip` so the absolutely-positioned hidden checkbox stays within the chip's stacking context.
- **Popover 0-draft state** — changed `renderDraftSection` in `poem.html` to hide the section when no drafts exist. **NOTE: This is immediately overridden by Phase 1.28 — see overrides below.**
- **ROADMAP bug log** — added "Same-name horses all render with the first horse's URL" to Bugs section of ROADMAP.md.

## Decisions made

- **Clover directed:** 0-draft popover should offer a quick-create form, not hide — `[Override: Clover directed]` supersedes the hiding fix just shipped in PR #39
- **Clover proposed, Claude approved:** Auto-save replaces explicit "Save Draft" — structural content saves immediately on add/remove events; drag saves with ~5-10s debounce; "Edit Details" replaces the Save Draft button for metadata
- **Clover proposed, Claude approved:** Editor page load with 1+ drafts shows "What draft would you like to edit?" picker; with 0 drafts drops directly into blank draft (no picker, less friction)
- **Clover proposed, Claude approved:** "Change Draft" button/picker IS the draft chip-switcher concept — same UI, refined label
- **Clover proposed, Claude approved:** Clear Stable / Clear Poem buttons replace the old "Clear" buttons; each shows a destination dialog (pasture / set loose / cancel; Clear Poem also offers "Send to stable")
- **Clover proposed, Claude approved:** "Edit Details" replaces "Save Draft" as the modal trigger label
- **Clover asked, Claude noted for roadmap:** Anonymous pasture + drafts in localStorage — yes, that's the anonymous flow already in place; the question of "why not let anon users have full feature access" answered by: nudge to register + sync-on-login already wires it up. No scope change.
