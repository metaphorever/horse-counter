## Work completed

### Phase 1.29 — DRY poem renderer

**`templates/macros.html`** (new file):
- `render_chip(h, linked=true)` — renders one horse chip with `poem-horse coat-X [rev] [famous-horse]` classes; uses `<a>` if linked and URL present, `<span>` otherwise
- `render_poem(lines, line_class='poem-line-out', linked=true)` — loops lines and chips using `render_chip`

**Template wiring:**
- `poem.html` — replaced inline chip loop with `render_poem(poem.lines)`
- `poem_queue.html` — replaced inline chip loop with `render_poem(sub.lines, line_class='queue-poem-line', linked=false)`; added `data-view-mode="pasture"` to `.queue-poem`; bumped padding-top to 26px and line-height to 2.4 to accommodate chip decorations
- `user_profile.html` — replaced bare `horse-chip` bio poem loop with `render_poem(bio_poem.lines)`; added `data-view-mode="pasture"` to bio wrapper

**Bug fixed:** bio poem on user profiles was rendering with bare `horse-chip` class — no coat/rev/famous styling. Route now enriches horse appearance before render.

**Collection pages:**
- `my_pasture.html`, `saved_horses.html` — switched from `horse-link` (JS-applied colors) to `render_chip` macro with `data-view-mode="pasture"` on the container; JS COATS hash removed
- Routes `me_pasture()`, `me_saved_horses()` enriched with coat/rev/is_famous
- `user_profile()` route also enriched for bio_poem lines

**CSS:**
- `[data-view-mode='pasture']` selector broadened from `.poem-view`-scoped — any container with the attribute activates pasture chip styling
- Turnover indent (`padding-left: 2em; text-indent: -2em`) added to screen `.poem-line-out`; overridden to zero in pasture mode (text-indent breaks inline-block chip flow — "horses follow different rules than text")
- Global view-mode preference: `base.html` now reads `poem-view-mode` from localStorage on `DOMContentLoaded` and applies it to all `[data-view-mode]:not(.poem-view)` elements — preference now propagates to bio poem, collection pages, and queue

**PRs:** #51 (main work), #52 (live-test fixes: massacre fix + bio/queue pasture mode), #53 (global mode preference)
