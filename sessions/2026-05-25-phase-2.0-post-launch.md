**Pasture infinite scroll popover bug fix**
- Root cause: `_horse_popover.html` wired click handlers via `querySelectorAll` at script-load time — only initial DOM chips got listeners; chips inserted by infinite scroll never did, so `<a>` elements followed their href to pedigreequery
- Fix: single event-delegated listener on `document` using `closest('.poem-horse[data-name]')`; catches all chips present or future
- Also: exposed `window.__mergeHorseStates(names)` for incremental state pre-loading; `loadMore` in `pasture.html` now calls it after each batch so ribbon/pasture-button state is correct on first open of a scrolled-in chip
