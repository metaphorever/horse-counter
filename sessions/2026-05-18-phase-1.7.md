## What shipped

**New DB tables:** `horse_occurrences` (poem/horse index for fast lookups), `pasture_horses`, `saved_horses` (per-user collections).

**New module:** `horse_collections.py` — `toggle_pasture`, `toggle_saved_horse`, `get_horse_states`.

**`poem_db.py`:** `populate_horse_occurrences()` called at poem save time; `get_poems_featuring_horse()` uses the index.

**Four new API routes:** `POST /horse/state`, `/horse/save`, `/horse/pasture`, `/horse/poems`.

**`templates/poem.html`** (new, replaces Phase 0.3 stub at `/p/<short_code>`): full pasture-mode permalink renderer — horse chips as focusable buttons with full tile styling (coat colors, head, tail, legs). Horse popover: name, PedigreeQuery link, blue-ribbon save toggle, add/remove from pasture, poems-featuring list. Focus-trapped, ESC to close, outside-click dismiss. Logged-out prompts with distinct copy per action.

**`app.py`:** `_tile_style()` Python helper registered as Jinja global (mirrors JS hash). `/p/<short_code>` now renders `poem_permalink.html`.

- `[confirmed]` Popover shape: name + PQ link + save toggle + pasture toggle + poems-featuring list. Confirmed in pre-phase design question.
