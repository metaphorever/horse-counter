## What shipped

**PR #62 ‚Äî Phase 1.13.1 ‚Äî Trust score system**

### Schema + migration
- `trust_score INTEGER NOT NULL DEFAULT 0` added to `users` via `db/seed.py` column migration (idempotent; runs on boot)
- New `admin_settings` table ‚Äî key/value store for admin-configurable thresholds; seeded with `auto_post_threshold = 0`
- `db/admin_settings.py` ‚Äî `get_setting`, `set_setting`, `get_auto_post_threshold` helpers

### DB helpers
- `db/users.py` ‚Äî added `update_trust_score(user_id, delta)`, `set_trust_score(user_id, score)`, `get_all_users()`

### Score updates at approval (`admin_poem_publish`)
- Captures original pending tag_ids from `poem_tags` before deletion
- Compares with admin's submitted `tag_ids`; `tags_edited = set(tag_ids) != original_tag_ids`
- After `approve_poem_submission()`: if poem has `author_user_id`, calls `update_trust_score(author_id, -1 if tags_edited else 1)`
- Anonymous/pseudonymous poems (no `author_user_id`) are unaffected

### Queue bypass at submit (`submit_poem_public`)
- `get_auto_post_threshold()` ‚Üí `None` if disabled, int otherwise
- `bypass_queue = post_as == 'account' AND current_user AND threshold is not None AND trust_score >= threshold`
- Bypass path: poem saved with `status='published'` (published_at stamped by save_poem_db), tags applied as 'approved', no submission row, response includes `"published": True`
- Default threshold = 0 ‚Üí all logged-in account-posting users bypass on first ship

### Admin user pages
- `GET /admin/users` ‚Äî all users with trust scores, threshold control form (save to `admin_settings`)
- `POST /admin/users/threshold` ‚Äî validates non-negative int or blank (disable); saves via `set_setting`
- `GET /admin/user/<id>` ‚Äî user detail: display name, slug, role, joined date, trust score with override form, published poems list
- `POST /admin/user/<id>/trust` ‚Äî manual trust score override via `set_trust_score`
- Queue card author names link to `/admin/user/<id>` when `author_user_id` is set; falls back to external link for pseudonymous authors
- "Users ‚Üí" nav link added to poem queue nav bar

### ROADMAP additions
- `Admin poem hide / delete` ‚Äî backlog entry with notes on natural home (permalink toolbar + queue card), cascade behavior, and connection to the pasture auto-populate hook
- `Admin account actions` ‚Äî backlog entry noting `/admin/user/<id>` as the home, suspension column design, delete-to-anonymous behavior

## Decisions

- **Clover, ok** ‚Äî `auto_post_threshold` defaults to 0 (open posting) on ship; focus is on easy adjustability, not the seed value
- **Claude design** ‚Äî tag edit detection: compare pre-deletion pending tag_ids set vs. submitted set; empty-to-empty counts as "no edits" (score +1)
- **Claude design** ‚Äî rejection does not affect trust score; score only moves on approval
