## What shipped (3 commits on the branch)

### Step 4 — Fancy skin: "the field is the surface" (`d09d344`)
- Key realisation: the site **already** paints the grass field on `<body>` (`.container` is transparent), so today's boxed Fancy look came from the `view-decorated` rule painting a *redundant* green box + brown border on each container. Making Fancy "field-is-surface" is therefore **subtractive**: containers go transparent and the existing body field shows through. This also *is* the bio-poem transparent-to-grass fix.
- Split the shared `view-decorated` container/panel rules → **`view-plain`** (Plain keeps its grass-box + cream note-boxes, unchanged) vs new **`view-fancy`** (transparent containers; title + poem float bare cream-with-halo; tags + footer become **inverted notes** — dark fill, tan text, gold hairline; non-horse text de-emphasised).
- CW pills and the admin tag editor keep their own light surfaces, so they stay legible on the dark note (admin guts stay utilitarian).
- Removed dead `.bio-line` / `.horse-chip` CSS (bio poem renders via `render_poem` since 1.29).

### Step 5 — Reader skin: print aesthetic on screen (`7ef6d6a`)
- Reader previously fell through to the base cream-card look. Now it ports the 1.11 print stylesheet to screen: off-white paper (#f5efe0), no field; Abril Fatface title, IM Fell English small-caps body with turnover indent, Playfair Display SC attribution + comma-separated tags, names marked by a thin underline (gap = separator). Masthead/colophon stay print-only.
- Footer buttons → **woodprint "stamped block"** borders (hard edge + offset solid shadow), press-in on `:active`, dim on the "Copied!" flash, and an **engaged/inked** state for the saved ribbon (pressed into its shadow, dark fill + light text) so "saved is saved" reads at a glance.

### Step 6 — Fancy mobile: confirm + minimal polish (`2d668d6`)
- Prototype review (with Clover's correction) showed the "long multi-line chip" fear was an artifact of an unrealistic 44-char test. **Registered names cap at 18 chars** (real Jockey Club rule the scraped data follows; not enforced in app code), and even the longest single no-space token (`horsewithalongname`) fits one chip-line on a 375px phone at the current scale. So **no chip shrink** — mobile Fancy already works.
- Minimal polish only: nudge the floating title to 1.45rem on ≤520px so it doesn't dominate a phone, + a little body top-padding. Chip scale untouched.

## Deferred / added to roadmap
- **SVG chip art system** (Phase 2, paired with walk-cycle/animation) — Clover raised CSS/Unicode "found objects" vs custom front/back SVGs + flexible middle. My take: SVG is the better long-term substrate, *because of motion* — the `♞`/`∫` glyphs are the real cross-platform drift, and gradient-slice legs can't walk; the deferred walk-cycle/roaming ambitions need SVG paths. Charm is preserved via deliberately-crude ("charming jank") art, not by keeping Unicode. The `render_chip` macro already isolates the chip, so the swap is contained and "refine once, correct everywhere" holds — which is exactly why deferring costs nothing. Keep CSS chips for now.
- **Style/font pass** gains a first-class item: **enlarge Fancy chips + harmonise the chip font with the display type** (currently Courier Prime monospace vs Playfair display). Pairs naturally with the SVG-chip conversation — same chip-craft decision.
- **Dark mode** (backlog) — the inverted tan-on-dark-brown note palette is a natural fit; Clover flagged it as a possible future direction. Not 1.12 scope.
