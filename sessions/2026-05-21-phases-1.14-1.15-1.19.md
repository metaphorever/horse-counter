## Decisions made

- **`saved_poems` table created via schema.sql `CREATE TABLE IF NOT EXISTS`** — existing DBs get the new table on next `init_db()` call (happens at app startup), no extra migration logic needed
- **Bio picker modal uses `.open` class pattern** — matches existing poetry.html modal convention, not `hidden` attribute
- **Report rate limit set to 3/hour** — same order of magnitude as the spec; simple floor; can be raised if legitimate traffic warrants
- **`/me/published` redirects to profile** — the profile page is the canonical published poems view; no separate list page needed
- **Poem save requires login; report does not** — poems can be reported by anyone (with IP rate limiting); saves are a personal collection and require an account
