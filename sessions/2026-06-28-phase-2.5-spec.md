## Decisions made (with Clover)

1. **Wander scrambles the poem by design.** Reading happens in formation; Set Loose
   breaks it, Return to Poem Formation restores it. The horses *are* the words, so
   roaming necessarily unreads the poem — that's the intended toy.
2. **Two orthogonal animations, named:** *Wander* (re-positioning chips out of inline
   flow — the bigger lift) vs *Walk-cycle* (leg motion — smaller). Built separately.
3. **Frame-based walk-cycle over procedural leg-pivots** (Clover's "complication").
   Decider: a frame cycle pays off *before* animation ships — a static horse renders
   a frame plucked from the cycle, so a herd stops looking like clones. The
   "complication" is just: extra leg frames + a few tail variations + head pivoting.
4. **Three-phase arc**, scariest part (sprite rework) first at lowest stakes:
   - **2.5.1 Static pose variation** — varied stance/head/facing, no animation/layout/JS.
   - **2.5.2 Walk-cycle** — frame animation + the View-menu tree + control strip.
   - **2.5.3 Wander** — roam the grass field (permalink-first) + gaits + avoidance.
5. **Arena** = the grass-textured viewport region; horses scroll *with* the page.
6. **Avoidance, not pathfinding** — soft repulsion from edges + UI rects (nav, strip,
   info/popover boxes). Open: avoid each other or pass through (lean: pass through).
7. **Control vocab** (supersedes Scatter/Reform): View → Fancy → {No Animations |
   Horse Animations}; when on, a try-on-style strip: **Hold Your Horses** (animate in
   place) · **Set Horses Loose** (roam) · **Return to Poem Formation** (reform) ·
   gaits **Graze / Walk / Trot**. Hold ≠ Return: Hold freezes-in-place, Return
   restores formation.
8. **Animation state persists as a view setting** (cookie/DB pref, like view-mode).
   `prefers-reduced-motion` forces effective "No Animations" (1.12 decoupling holds).
9. **Static poses = rule-constrained picks** from the leg frames ("≥3 of 4 hooves on
   the ground"–type rule, tunable); curated stance set as the safe default.
10. **Randomized per render** (pose + direction), not hashed — random costs the same
    and Clover wants fresh poses each load. Server-side, no-JS-friendly.
11. **Head** = one image pivoted across ~6 angles (up↔down), not frames.
    **Tail** = 4–6 swish frames (start fewer; static picks one).
12. **Static variety is global** (a `horse_svg()` macro change → every server-rendered
    Fancy surface). Only Wander is permalink-first. JS infinite-scroll pasture const
    stays single-pose until later.
13. **Carrot herding toy** — fun future idea, the one place real pathfinding would
    live; explicitly deferred past 2.5.3. Logged to backlog.
14. **Spec structure:** umbrella + full 2.5.1 only; spec 2.5.2 / 2.5.3 after each
    prior phase ships and its lessons are in hand.
15. **Numbering:** grouped as the 2.5 family to avoid collisions with the loose
    2.7/2.10/2.13 references already in the backlog/open-questions. Renumberable.
