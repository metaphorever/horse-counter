## What shipped

**Phase 1.26 — Horse search area rework**

- **Short names → button** — `Browse short names` link removed from the old inline hint block; replaced by a `Short names` button in a new quick-picks row at the top of the search panel. Same `doShortNames()` call, now a proper button.
- **Random moved to search panel** — `+ random` button removed from the Stable panel's `stable-controls`. New `Random` button in the quick-picks row; `doRandom()` rewritten to load per-page-count results into the search display area instead of adding directly to stable. Random cap raised from 20 to 50 to match max per-page setting.
- **My pasture button** — new quick-pick button (logged-in users only). `doPastureHorses()` calls new `/poetry/pasture-horses` POST route, which fetches the user's pasture horses sorted alphabetically (via `list_pasture_horses`) and returns them shaped for the search result renderer. Pagination handled by existing client-side pager.
- **Help modal replaces inline hint** — `search-hint` div (wildcard syntax examples + short names link) removed. Replaced by `? How it works` button at the bottom of the search panel. Opens a modal covering the full pipeline: search → stable → poem → post, with wildcard syntax moved to a footnote block.

**Files changed:** `app.py`, `templates/poetry.html`, `static/style.css`
