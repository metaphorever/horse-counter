### Decisions settled at build (spec amendment)
- **Render technique:** ONE `<svg>` per chip, all parts in one coordinate space
  (replaces the spec's original floating-`<use>`-layers idea). Needed so one
  `filter:drop-shadow` covers the whole silhouette and the famous shimmer can
  sweep it continuously. **Name stays HTML** (`.hz-word`) on top → auto-width,
  `min-width:6ch`, popover, accessibility all unchanged.
- **No-JS core + JS shimmer:** the horse (body/legs/head/tail/eye/shadow/far-leg
  shading) renders with no JS; the whole-horse famous shimmer is a light
  progressive-enhancement script (Clover: "JS as a treat"), reduced-motion-aware.
- **Eye A** (round white sclera + black pupil overlay) over Clover's literal drawn
  eye — more legible/characterful; + dark nostril. Both live in the head
  `<symbol>` as the single tweak point.
- **Far-leg shading Fore Y / Hind X** (depth cue) — fixes the jumbled Illustrator
  leg ids: shaded back legs = front-inner (`lfn`) + hind-outer (`lhf`), drawn
  behind the barrel; lit front legs = front-outer (`lff`) + hind-inner (`lhn`).
- **Per-coat tilt dropped** (belongs to the animation layer, not coats).
- **No pre-rendering** (proportional Fancy font ≠ char-count; and it would
  combinatorially explode once the walk-cycle lands — the live single-SVG keeps
  legs addressable so the walk-cycle stays procedural + length-independent). Also
  corrected the spec's stale "monospace 6ch self-adjusts" note.

### Build

### Live verification (Clover) — partial
Generally good. **Safari iOS: good.** Safari macOS: not yet tried. Three issues:

1. **Profile bio horses render "sage"** (fallback colour, not per-coat). **Clover
   confirmed pre-existing** — the coat colouring was off on `/u/<slug>` bios
   before 2.4; the old glyph chip just hid it (transparent-ish), the full SVG body
   renders the fallback colour outright. Diagnosis: bio chips on the profile path
   aren't resolving `--bg` (coat var). `bio_poem` IS coat-enriched (get_poem_by_id
   → _row_to_poem → _enrich_lines) and `view_decorated` is global, so the miss is
   elsewhere in the `.profile-bio-poem` render/context — **investigate next
   session**, treat as a profile bug, not a 2.4 regression.
2. **Infinite-scroll pasture chips are text-only.** Root cause confirmed:
   `templates/pasture.html` `chipHTML()` builds appended chips client-side from
   the `/pasture/more` JSON and emits `<a/span class="poem-horse …">name</a>`
   **without** the `horse_svg()` SVG. Initial (server, macro) chips are fine.
   Fix: have the JS emit the same SVG (cleanest: clone `.hz` from an existing
   server-rendered chip — single source — or have the endpoint/macro supply it).
   Check the saved-horses / my-pasture "load more" paths for the same pattern.
3. **Name sits a touch high** — Clover to tune `.hz-word` / `.hz` `translateY`
   manually (tunable knobs).
