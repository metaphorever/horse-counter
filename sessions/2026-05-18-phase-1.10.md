### Tagging bug fixes

Three bugs diagnosed and fixed:

1. **Tags never showed on published poems** — `poem_submissions.approve()` published poems but left `poem_tags` rows as `pending`. Fixed: bulk-`UPDATE poem_tags SET status = 'approved'` for the poem's pending rows at publish time.

2. **Tag editor appeared functional but changes didn't stick** — `applied_tag_ids` was built from approved tags only, so submitted (pending) tags showed as unchecked. Admin saved without checking → approved set cleared → nothing visible. Fixed: admin view of `poem_permalink` now includes pending `poem_tags` in `applied_tag_ids` so the editor pre-checks the submitter's choices.

3. **Schema sync** — `db/schema.sql` `tag_categories` was missing the `admin_only` column (present in `seed.py` migration, absent from the DDL). Updated the schema for fresh installs; migration remains as belt-and-suspenders for existing DBs.

4. **Admin tag UX confusion** — added a hint to `templates/admin_featured.html` explaining that admin tags are applied per-poem from the poem's permalink page, not from the Featured & Tags admin page.
