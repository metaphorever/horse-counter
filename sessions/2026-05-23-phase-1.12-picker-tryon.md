Short fix-forward triggered by Clover's live testing of the first-run picker. The foundation shipped the picker so that *any* interaction (including the first mode pick) wrote the cookie and dismissed the strip. Clover wants the strip to be a **playground**: click between modes and watch the site transform, with the prompt **staying put** until an explicit Save or dismiss.

## Change

Decoupled two signals that the foundation had conflated:
- **active mode** (`view_mode` cookie / `poem_view_mode` pref) — set on every mode click ("try it on").
- **picker decided** (`view_decided` cookie + mirrored pref) — set *only* by Save or dismiss; this is what hides the first-run strip.

- **`app.py`** — `resolve_view_mode()` → `resolve_view()` returns `(mode, decided)`, reading prefs once. `/set-view-mode` now takes independent `mode` (try-on) and `decide=1` (Save/dismiss) inputs; mode clicks no longer dismiss. `view_decided` added to `_USER_PREF_WRITES`. One-shot `session['view_picker_saved']` drives the post-decision confirmation (popped in `inject_globals`).
- **`base.html`** — strip stays while `not picker_decided`; mode buttons are try-on (highlight the active mode), plus a **"Save view setting"** button and an X (both `decide=1`). After deciding, the strip is replaced (once) by: "You can always change your view settings from the ⊙ menu[ or your Settings page]."
- **`profile_edit.html`** — the settings mirror sends `decide=1` (a deliberate pick in settings counts as decided).
- **`style.css`** — minimal styling for the Save button + message (deferred to the style pass otherwise).

## Behavior now

- Click Fancy/Plain/Reader in the strip → site reloads in that mode, **strip stays**, clicked mode highlighted. Sample freely.
- **Save view setting** or **X** → strip goes away for good (cookie + pref), replaced once by the "change it from the menu" message.
- Never decide → strip keeps showing on display surfaces.
- Logged-in: try-on writes the pref but does **not** dismiss; Save/dismiss sets `view_decided`; the `/me/profile` mirror sets both.

Each mode click is a full server reload (no-JS, server-resolved). A JS "instant snap" enhancement is possible later if the reload flash feels clunky — not built.
