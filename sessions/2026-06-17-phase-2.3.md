### Design (spec → committed before code)
`spec/phase-2.3-bluesky-composition.md`. Started as Clover's two questions about
Bluesky crossposts: (1) fit a poem sample into the 300-char limit; (2) how to be
discoverable as a zero-follower account. Grew a third feature mid-discussion when
the CW-label plumbing reminded Clover that Tumblr crossposts also drop the poet's
own site tags. All three share one data change, so scoped together.

Key decisions with Clover:
- **Bluesky sample body.** Card-only permalink (frees ~25 chars; link cards don't
  count against the 300). Whole-horse truncation with a `[…]` marker; degenerate
  first-line case cuts at a horse boundary. Budget to **290** with `len()` — the
  10-char margin covers the grapheme-vs-codepoint gap without a dependency.
- **Discoverability.** Research (Bluesky public API, `getPopularFeedGenerators`)
  showed the horse world fragments by sub-interest. Clover chose `#horses`
  (affection/art crowd) over `#horseracing` (industry/betting) despite the horses
  being racehorses — vibe match over literal match. The biggest poetry feed (497
  likes) is **text-based** (scans for "poem"), so the header lands it for free,
  freeing the tag budget. Final set: `#poetry`/`#horses`/`#PoetHorse`, with
  `#micropoetry` swapping in for `#poetry` on ≤140-char bodies (cutoff is a
  tunable constant; Clover confirms live before posting at volume). Tags render
  as real **facets** (plain `#text` via API isn't tappable/indexable).
- **Content label.** Site `sex` CW → Bluesky `sexual` self-label (suggestive,
  no-image; `porn` is for explicit images). Safety net under Clover's manual
  final-say, not a gate.
- **Tumblr site tags.** Splice the poet's tags between the identity tags and
  trailing boilerplate (before `counting-horses`), `label.lower()` rendered (so
  multi-word tags read with spaces), CW category prefixed `cw `.

### Build
- `db/crosspost.py` — `get_pending()` now attaches `tags_for_poem(poem_id)` as
  `item['tags']`. The single shared data change; feeds both consumers below.
- `poetry.py` — `site_tags_for_crosspost()`: renders a poem's tags for Tumblr
  (`label.lower()`, `cw ` prefix on content-warnings, admin-only dropped, order
  from `tags_for_poem`'s category sort).
- `app.py` — `_build_crosspost` splices those before `counting-horses`, deduped.
  `_build_bluesky_post` rewritten to return ingredients (header, per-line horse
  displays, reach tags, self-label, card fields) + the Bluesky policy constants
  and `_sanitize_bsky_tags`. Dispatch call simplified to `post_poem(**…)`.
- `bluesky.py` — `_compose_body()` does the budget-aware whole-horse sampling;
  `post_poem()` rewritten to build text+tags via `TextBuilder` (tag facets),
  attach `langs=['en']` and an optional `SelfLabels`, and send via
  `app.bsky.feed.post.create` (this atproto version's `send_post` has no `labels`
  param, so the record path is used directly).

### Spec bug found during build (fixed in place)
Spec listed "card alt text" as a freebie. AT Protocol **external/link cards have
no `alt` field** — only image embeds do. Title + description serve that role.
Dropped from the spec; alt text returns with image cards (Playwright, out of
scope). `langs=['en']` remains the one freebie.
