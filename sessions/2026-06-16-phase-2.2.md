### Design (spec → committed before code, per rule 2)
`spec/phase-2.2-bluesky.md`. Key decisions with Clover:
- **Two targets kept** (Tumblr + Bluesky). Clover proposed collapsing the poem
  dispatch to a single universal action instead of mirroring Tumblr's three
  modes — accepted. Poem path is now **direct live posting only**.
- **Partial-failure handling** was the one wrinkle a universal button can't
  dodge. Resolved with two nullable per-platform status columns
  (`tumblr_status`/`bluesky_status`: NULL → posted/failed/skipped). An item
  resolves out only when both are posted/skipped; a failure stays pending so
  re-dispatch ("Retry Crosspost") re-fires only the unfinished platform.
  Steelmanned the simpler one-status-column idea and rejected it: without a
  per-platform marker a half-failed crosspost either looks done or forces a
  double-post.
- **Counter feature untouched.** Clover clarified the "count horses in text"
  flow still wants Tumblr queue/draft (driven from PA today, to be revamped
  later). Scope narrowed to the *poem* path only; `queue_handler.submit_post`'s
  action selector is left intact and the poem path simply always calls it with
  `action='post'`.
- **Text + link-card now**, image cards deferred to the Playwright work.
- Account `poethorse.bsky.social`; post text `"{title}" a poem of {count}
  horses by {author}`; app password lives only in the VPS `.env`.

### Build
- `bluesky.py` — `BlueskyManager`, atproto app-password auth, **lazy** atproto
  import (so a missing dep / bad creds can't break boot or Tumblr posting — only
  surfaces as a Bluesky dispatch error). `post_poem` sends text + an
  `AppBskyEmbedExternal` link card.
- `config.py` — `BLUESKY_HANDLE` / `BLUESKY_APP_PASSWORD` from env.
- `db/seed.py` — added the two status columns via the existing
  `_COLUMN_MIGRATIONS` pattern.
- `db/crosspost.py` — reworked: `get_pending` uses the not-resolved-on-both
  rule; `mark_platform(cq_id, platform, status)` (column whitelisted); `skip`
  resolves both.
- `app.py` — instantiate `bluesky`; `_build_bluesky_post`; dispatch route
  collapsed to one action firing all connected platforms with per-platform
  result recording and a per-platform flash summary.
- `templates/admin_crosspost_queue.html` — single Crosspost/Retry button,
  per-platform `✓/✗/—` status, both connection banners.
- `requirements.txt` — `atproto>=0.0.46,<1`.
