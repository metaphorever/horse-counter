## What shipped

**PR #38 — Phase 1.27 (merged this session)**

Full draft-centric stable model. Draft belongs to a specific poem, not a global per-user pool. Removes the temporary `stable_horses` table and `/me/stable/*` routes added in PR #37 (superseded by this design).

### db/drafts.py (new)
CRUD helpers: `save_user_draft`, `list_user_drafts`, `get_user_draft`, `add_horse_to_draft_stable`, `delete_user_draft`. Separate from `queue_handler.py`'s ephemeral file-based drafts (those are for the Tumblr submission queue and are untouched).

### db/seed.py
Extended `drafts` table via `_COLUMN_MIGRATIONS`: `stable_json`, `submitter_name`, `submitter_tumblr`, `inspired_by_text`, `inspired_by_url`, `tag_ids_json`, `created_at`.

### db/stable.py — DELETED
No longer needed. `stable_horses` table remains in schema (DROP handled lazily — no data, harmless).

### app.py
- Removed `/me/stable/add`, `/me/stable/remove`, `/me/stable/clear` routes
- Stripped stable sync from `/me/sync` (now preferences only)
- `poetry_editor()` no longer loads stable from DB for Clerk users (starts empty)
- Added `/me/draft/save` (create/update), `/me/draft/list` (JSON for chip picker), `/me/draft/stable/add` (add horse to specific draft's stable), `/me/draft/delete`
- `/me/drafts` now renders real list via `me_drafts.html`

### templates/me_drafts.html (new)
Drafts list page: title, horse count, Resume editing link, Delete button (AJAX, removes item without reload).

### templates/poetry.html
- **Save Draft button** added to poem actions area; disabled for anonymous users (tooltip: "Sign in to save drafts"); disabled when poem is empty for logged-in users
- **Draft modal**: same fields as publish modal (title, name, tumblr, inspired-by, tags); saves to SQLite via `/me/draft/save`; `draftId` state tracks which draft is active (re-saves update in place)
- **Stable ops simplified**: `addToStable`, `removeFromStable`, `clearStable`, `removeTile`, `dropHorseOnLine`, `removeHorseFromSource` no longer call `/me/stable/*` for Clerk users — all paths collapse to in-memory + `saveDraftLocal()` for anonymous / no-op for logged-in
- **`horse-draft` localStorage shape**: `{lines, stable, title, submitter_name, submitter_tumblr, tag_ids}` replaces the old `horse-stable` (stable-only) shape
- **Migration**: `loadDraftLocal()` detects old `horse-stable` or `horse-pasture` keys and migrates them into the new shape on first access
- **`syncLocalToAccount()`**: now creates a SQLite draft from any non-empty `horse-draft` on first login, then clears all legacy localStorage keys
- **`renderTagSections` refactored** to `_renderTagSectionsInto(hostId, tagSet, onRerender)` so both the publish and draft modals share the same tag-chip rendering logic without duplication

### templates/poem.html
Horse popover gains a `popover-draft-section` div. When opened by a logged-in user:
- Draft list is lazy-fetched from `/me/draft/list` (cached per page load)
- 0 drafts: "start a poem first" note with link to `/poetry`
- 1 draft: direct "Add to [title]" button
- 2+ drafts: inline picker with one button per draft
- Each button POSTs to `/me/draft/stable/add` and updates its own label on success

## Decisions made

All design decisions were locked in the prior session (2026-05-19-phase-1.27-design.md). No new design calls this session — implementation only.

**`lines` auto-save for anonymous users** · `[Claude decision]`
`saveDraftLocal()` is called on all stable mutations. Also called from `clearPoem()`. Not called from `addLine()` (empty line not meaningful). Lines persist via `dropHorseOnLine` and `removeTile` which both call `saveDraftLocal()`. This is sufficient coverage.

**`renderTagSections` refactor** · `[Claude decision]`
Rather than duplicating ~40 lines for the draft modal, refactored to accept `hostId` and `tagSet` parameters. Both modals share the implementation cleanly.
