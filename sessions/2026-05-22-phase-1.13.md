### Phase 1.13 — Admin moderation queue overhaul

`/admin/poem-queue` redesigned:

- **Horse chip render**: each submission now shows styled chips (coat/rev/famous classes applied via the same `horse_appearance()` enrichment as the permalink view) instead of plain links.
- **Tag review — chips-first UX**: submitter's pending tags shown as removable chips by default. Each chip has an × button that unchecks the corresponding hidden checkbox. "Edit tags" reveals the full checkbox picker (all categories, submitter tags pre-checked) for poems that need more curation. "Done editing" rebuilds the chip display from checkbox state.
- **Publish sets tags explicitly**: route now clears all poem_tags (pending + approved) for the poem before applying the admin's selection as approved, then calls approve. No more auto-approve blindly passing through pending tags.
- **Reviewer ID wired**: both approve and reject now pass the Clerk user ID as reviewer_id (was None before).
- **Full preview link**: each card has a "Full preview →" link to the draft permalink for when you want to see the full rendering before approving.
- **Non-nested form structure**: publish form wraps the tag review section; publish button outside via `form=` attribute; reject is a separate sibling form. Valid HTML5, no nested forms.
