### Hold 1 — profile bio horses rendered link-green (PR #76)
Root cause was **not** the `--bg` resolution the prior log suspected — coats *do*
resolve. The chip `<a class="poem-horse">` is a link inside `.profile-page`, so
`body.view-fancy .profile-page a:not(.btn-*)` (specificity 0,5,2) overrode
`body.view-fancy .poem-horse { color: var(--bg) }` (0,2,1), forcing the chip's
`color` to the panel link colour (`#c8e8a0`). The `currentColor` silhouette parts
(head/tail/barrel/foreground legs) inherited it; the `.hz-shade` legs, `.hz-word`
name, and fixed-fill eye/nose were spared because each sets its own colour — which
is exactly the symptom Clover reported (only the body parts went green, matching
the link colour). Pre-existing; the full SVG body just made an always-present
mis-colour glaring. **Fix:** add `:not(.poem-horse)` to the panel link rule
(`style.css`). Works regardless of the bio's background, so the dark-box-vs-grass
profile redesign Clover floated is decoupled — left as a future intentional design
pass (Clover: not married to the redesign, will plan it properly if/when).

### Hold 2 — infinite-scroll pasture chips were text-only (PR #76)
`templates/pasture.html` `chipHTML()` built appended chips with bare text — no
`horse_svg()` SVG, no `.hz-word` span. **Fix:** import `horse_svg` and emit it once
as a JS constant (`HORSE_SVG`), wrapping it + `.hz-word` exactly like `render_chip`
— single source of truth (the macro stays canonical). Checked saved-horses +
my-pasture: server-rendered only, no JS append → not affected.

### Hold 3 — positioning knobs (PR #77)
Clover's live tuning surfaced two *real bugs* behind the "name too high" symptom:
1. **Body offset ignored on mirrored horses.** `.hz` and `.rev .hz` each hardcoded
   `translateY(-40%)`; the more-specific `.rev` rule meant changing the base only
   moved left-facing horses. **Fix:** single `--hz-y` var both facings read.
2. **Name offset was a silent no-op.** `.hz-word` was `display:inline`, and
   `transform` is ignored on non-replaced inline elements — so `translateY` never
   applied (the "+4px" from the build session had never done anything). **Fix:**
   `display:inline-block`. Both are now annotated `TUNE HERE` single knobs.

Final tuned values (Clover, live): `--hz-y: -50%`, `.hz-word translateY(0px)`.
