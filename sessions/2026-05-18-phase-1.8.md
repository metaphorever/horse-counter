## What we did

**Scoping conversation (start of session)** — The original 1.8 scope (Featured as separate curation layer, Browse with pagination/sort/filter, Random redirect) expanded during conversation. What shipped:

**Two-tier tag system** — Core architectural addition. `tag_categories` gets an `admin_only` column (column migration in `db/seed.py`). Public tags (admin_only=0) are the existing system. Admin-only tags (admin_only=1) are invisible to non-admins and exist for internal curation signals. The two tiers share the same taxonomy structure.

**Featured sections** — New `featured_sections` table in `db/schema.sql` (`CREATE TABLE IF NOT EXISTS`, idempotent). Each row pins a tag (public or admin-only) to a slot on `/featured` with a custom label, sort_order, and active flag. Admin curates these from `/admin/featured`. The `/featured` page renders each active section as a labelled group of poems tagged with that tag.

**Browse** — Full replacement of the 500-poem flat dump. Pagination (20/page), sort (newest/oldest/most horses/fewest horses), tag filter (all active public tags, grouped by category), "based on existing works" attribution filter. New `browse_poems()` and `count_browse_poems()` in `poem_db.py`.

**Random** — `/random` now redirects to a randomly selected published poem. If no poems exist, flashes and goes to `/browse`.

**Per-poem admin tag editor** — On the poem permalink, admins see:
- The normal public tag chip display, with an "Edit public tags" `<details>` expander below it containing checkboxes for all public tags
- A separate "Admin tags" section (admin-only) with its own "Edit admin tags" expander
- Tier-aware POST to `/poem/<short_code>/tags?tier=public|admin` — preserves the other tier's tags on each save

**Admin featured management page** (`/admin/featured`) — Three panels:
1. Featured sections table: label, tag, sort_order, active — editable inline, removable
2. Admin-only tag categories: create categories, add tags within them
3. Public tag categories: read-only list, with "add tag" form per category (full taxonomy editor stays in 1.4)

## Decisions

- `[confirmed]` Two-tier tag system: same `tag_categories` table, `admin_only` flag separates tiers. Not a separate table.
- `[confirmed]` Featured page driven by `featured_sections` table — any tag (either tier) can be featured; custom header per section, sort_order controlled by admin.
- `[confirmed]` Per-poem tag editor uses `<details>`/`<summary>` — no additional JS. Two separate forms (one per tier). Tier-aware backend preserves the unedited tier.
- `[confirmed]` Phase 1.4 (full public tag taxonomy editor) stays deferred. Admin can add tags to existing categories from `/admin/featured`; renaming/merging/approving user suggestions is 1.4.
- `[confirmed]` Tag chips on browse page link to `/browse?tag=<slug>` rather than a stub `/tag/<slug>` (per-tag landing pages are Phase 2).
